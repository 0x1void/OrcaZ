### 5) Zero‚ÄëTrust Rules üõ°Ô∏è

* **Default inter‚ÄëVLAN:** deny all; allow only explicit flows.
* **W-Link overlay:** only authenticated admin reaches management (pfSense, AD, Zabbix, Wazuh, GLPI, WinSrv, Vault).
* **OpsNet ‚Üí CoreNet:** allow DNS 53, LDAP/LDAPS 389/636, Kerberos 88/464, WinRM 5985/5986 (if used).
* **ClientNet ‚Üí Vault:** allow SMB 445 (encryption required); allow AD join; block the rest.
* **Perimeter ‚Üí Lab:** no routes.
* **DNS egress:** servers must use Sentinel; drop raw 53/853 to Internet.
* **Host firewalls:** local default‚Äëdeny; allow only from **W-Link** and designated VLANs.

### 6) Hardening üîí

Minimal OS; timely patches; SSH keys only (`PermitRootLogin no`), MFA where supported.
pfSense: HTTPS admin; strong creds; config backups; pfBlockerNG optional.
AD/DNS: secure dynamic updates; password/lockout policy; time sync; delegated groups.
Vault: ZFS snapshots; **SMB encryption required**; least‚Äëprivilege shares; audit logs ‚Üí Wazuh.
Sentinel: overlay bound to internal; Pi-hole DoH/DoT upstream; rotate pre‚Äëauth keys.
All hosts: Zabbix + Wazuh agents; local firewall default‚Äëdeny; central logging.

---

### 7) Backups & Recovery üíæ

* **3‚Äë2‚Äë1** rule (3 copies, 2 media, 1 offsite).
* **Configs:** pfSense XML, Samba AD state, GLPI/Zabbix/Wazuh DB dumps.
* **Data:** ZFS snapshots (hourly/daily/weekly) replicated off‚Äëbox.
* **Drills:** quarterly restore test (one VM + one dataset).

### 8) Build Steps üß∞

1. **Sentinel** (no GUI): Debian/Arch ‚Üí **Headscale** (users, keys, ACL) + **Pi‚Äëhole** (DoH/DoT).
2. **Vault**: Debian ‚Üí **OpenZFS** (`tank`, `tank/shares`, `tank/backups`); **SMB 3.1.1 encryption**; optional NFS; set `zfs_arc_max‚âà2 GB`.

6. **Ops VMs**: **Zabbix**, **Wazuh**, **GLPI**, **WinSrv** on VLAN20; auth to AD/DNS.
